YouTubeチャンネル運営のためにAI音楽サービスを片っ端から試した結果、SunoとSoundrawに落ち着きました。両方に課金して半年使い込んだので、技術的な使い分けポイントと、実際のパイプラインに組み込んだコードまで書きます。高校生でも使えるくらい簡単なので、副業や個人開発でBGMが必要な人の参考になればうれしいです。

## どっちが何屋なのか

ざっくり整理するとこうなります。

| 観点 | Suno | Soundraw |
| --- | --- | --- |
| 出力形式 | ボーカル付きフルコーラス | インスト (歌なし) |
| 学習データ | 楽曲データ (詳細非公開・訴訟係争中) | 自社制作音源のみ |
| 構成編集 | ガチャ再生成のみ | ブロック単位で編集可 |
| 商用利用 | 有料プランで可 | 有料プランで可 (権利関係明確) |
| 個人向け月額 | 約¥1,500 | 約¥2,550 |
| API | 非公式のみ | 公式なし (DL自動化は規約注意) |

Sunoは「曲を作るAI」、Soundrawは「BGM職人のためのDAW的ツール」。これを最初に知っておくと迷わないです。

特にSunoは2026年初頭時点でUMG等のレコード会社から著作権訴訟を受けていて、商用利用するなら最新の規約を必ず確認した方がいいです。僕も収益化チャンネルではSoundraw一択にしています。

## 実際のパイプラインに組み込んだ例

YouTubeのStoic系Shorts (60秒) を量産するときの構成です。BGMはSoundraw、ナレーションはedge-tts、字幕はffmpegで焼き込み。

```bash
# 1. Soundrawで生成した曲をDL (手動DLしたものを使う)
INPUT_BGM="bgm/stoic_dark_60s.mp3"
NARRATION="voice/narration.wav"
OUTPUT="output/short_001.mp4"

# 2. 音量バランス調整: BGM -24dB / ナレ -6dB
ffmpeg -i "$NARRATION" -i "$INPUT_BGM" \
  -filter_complex "[0:a]volume=-6dB[a0];[1:a]volume=-24dB[a1];[a0][a1]amix=inputs=2:duration=first" \
  -c:a aac -b:a 192k mixed.aac

# 3. 映像と合成
ffmpeg -i background.mp4 -i mixed.aac \
  -c:v copy -c:a aac -shortest "$OUTPUT"
```

この `-24dB` と `-6dB` の比率は実測で決めました。最初 `-12dB / -3dB` でやったらBGMがうるさくてナレーションが埋もれ、視聴維持率が35%台で頭打ちに。比率を4:1にしたら45%まで上がりました。地味だけど効きます。

## ループ素材としての扱いやすさ

SoundrawはYouTube用の長尺BGM動画でも使えます。たとえば「4時間睡眠用ピアノ」みたいな動画を作るとき、Soundrawで3〜5分の曲を生成して `ffmpeg` でループ連結します。

```bash
# 3分の曲を80回ループして4時間に
ffmpeg -stream_loop 79 -i bgm_3min.mp3 -c copy bgm_4hour.mp3

# クロスフェードで継ぎ目を消したい場合
ffmpeg -i bgm_3min.mp3 -af "afade=t=in:st=0:d=2,afade=t=out:st=178:d=2" faded.mp3
ffmpeg -stream_loop 79 -i faded.mp3 -c copy bgm_4hour_smooth.mp3
```

Sunoは2分前後でブツ切れになり、しかも歌が入っているのでループ用途には全く向きません。これは試して初めて知った時、ちょっと驚きました。「同じAI音楽だから同じように使えるだろう」と思い込んでいたので。

## 「盛り上がり位置」を制御できる強さ

Soundrawの最強機能はこれです。曲のブロック (Intro / Verse / Chorus / Bridge / Outro) を秒単位で動かせる。動画編集の「ここで盛り上がってほしい」を後から指定できるのは作業効率が桁違いです。

たとえば60秒のShortsで「45秒目に盛り上がりたい」場合:

- 0-10s: Intro (静か)
- 10-40s: Verse (ナレーション乗せる用)
- 40-55s: Chorus (オチで盛り上がる)
- 55-60s: Outro (フェードアウト)

これをスライダーで組み立てられます。Sunoだと何回もガチャを回して理想の構成を引くしかなく、再現性がゼロです。

## 数値で見る制作効率

実測した1本あたりの制作時間です (60秒Shorts、BGM選定のみ)。

| 工程 | Suno | Soundraw |
| --- | --- | --- |
| 生成回数 (理想の曲が出るまで) | 平均7回 | 1-2回 |
| 1曲あたり生成時間 | 約40秒 | 約15秒 |
| BGM決定までの総時間 | 約8分 | 約2分 |
| 動画尺合わせ調整 | 別途ffmpeg編集必要 | エディタ内で完結 |

Sunoの「ガチャ感」は遊びには楽しいんですが、量産パイプラインに組み込むと致命的に遅いです。

## 失敗から学んだこと

Suno無料プランで作った曲をYouTubeに上げたらClaim (権利申し立て) が来たことがあります。よく規約を読むと、無料プランの曲は商用利用NGかつSuno側にも権利が残る建付けでした。最低でも有料プラン契約後の曲を使う、または商用利用は基本Soundrawに統一する、というルールに変えました。

もう一つの失敗は、Soundrawで生成した曲を自分のローカルキャッシュに溜め込んで再利用していたら、契約解除後に使えなくなったこと。Soundrawは「契約期間中に生成した曲は契約後も使えるが、契約解除後の新規生成は不可」という仕様で、ライセンス証明書を曲ごとにダウンロードしておく必要があります。これも公式に小さく書いてあるだけだったので、最初は気づきませんでした。

## 結論: 用途で完全に分ける

| やりたいこと | 選ぶべき |
| --- | --- |
| 動画BGM (商用) | Soundraw |
| 長尺ループBGM | Soundraw |
| TikTok/Shortsで主役の歌 | Suno (規約最新版を要確認) |
| 個人の遊び・オリジナル曲 | Suno |
| 副業の量産パイプライン | Soundraw |

両方契約しても月¥4,000程度。1本でもバズれば回収できる金額なので、本気でやるなら両方持っておくのが正解だと思います。ただし収益化を考えるなら、権利関係がクリアなSoundrawを軸に据えるのがおすすめです。

最新の料金・規約は必ず公式サイトで確認してから契約してください。AI音楽サービスは1ヶ月で規約が変わることもザラなので、油断は禁物です。
